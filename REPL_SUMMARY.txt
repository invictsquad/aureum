═══════════════════════════════════════════════════════════════════════════════
                          TECHNICAL ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

EXECUTION FLOW:

1. User types: input = tensor(shape=[1024], type=int16)
   ↓
2. Lark parser validates syntax
   ↓
3. Creates Tensor object (Python) with metadata
   ↓
4. Calculates memory: 1024 * 2 = 2048 bytes
   ↓
5. Shows visual feedback

6. User types: result = input * weights[::512]
   ↓
7. Parser extracts: input_var, weights_var, scale
   ↓
8. Calls Rust kernel via FFI (ctypes):
   
   Python:
   ───────
   kernel.pack_ternary(weights)      # Compresses to 2 bits/weight
   kernel.bitnet_infer(input, packed, scale)  # Executes inference
   
   ↓ (via ctypes)
   
   Rust (ffi.rs):
   ──────────────
   aureum_pack_ternary(...)          # Receives C pointers
   aureum_bitnet_infer(...)          # Executes natively
   
   ↓
   
9. Returns result (i64) to Python
   ↓
10. Shows result and statistics
